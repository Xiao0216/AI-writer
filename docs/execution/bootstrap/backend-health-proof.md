# Quest 1.3 后端健康检查验收模板

## 目标

证明 `apps/api` 已经可运行，并且有考官可见的黑盒接口。

## 学生需要提交的证据

### 终端证据

- API 启动日志截图

### 命令行证据

- 执行 `curl http://localhost:3001/health` 的终端截图

### 返回结果证据

- 返回 JSON 中必须看到：
  - `status`
  - `ok`

## PM 黑盒验收动作

- [ ] 启动 API 服务
- [ ] 请求 `/health`
- [ ] 确认返回结果是 JSON，不是 HTML 或报错

## 常见错误

- 服务起了但没有接口
- 健康检查返回不稳定
- 接口地址和课程端口约定不一致
