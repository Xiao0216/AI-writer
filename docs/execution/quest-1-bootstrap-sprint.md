# Quest 1 工程底座冲刺

## 目标

把前后端、数据库和测试环境统一跑起来，为后续开发 Quest 提供稳定底座。

## 必交付物

### 1. 根目录统一脚本

至少存在：

- `dev`
- `build`
- `lint`
- `test`

### 2. 前端可访问

要求：

- `apps/web` 能启动
- 浏览器打开 `http://localhost:3000` 有明确页面反馈

### 3. 后端可验证

要求：

- `apps/api` 能启动
- `http://localhost:3001/health` 返回 JSON

### 4. 基础设施可运行

要求：

- PostgreSQL 运行中
- Redis 运行中

### 5. 测试环境统一

要求：

- 存在 `apps/api/.env.test`
- 存在 `apps/web/.env.test`
- 可通过统一脚本启动 test 环境

## 验收动作

### PM 检查动作

- [ ] 在根目录执行脚本查看统一入口
- [ ] 打开前端页面
- [ ] 访问后端健康检查
- [ ] 查看基础设施运行状态
- [ ] 检查测试环境文件存在

## 工程原则

- 先把系统跑起来，再继续业务开发
- 当前阶段不做业务功能扩展
- 所有后续 Quest 都默认建立在这套底座之上
