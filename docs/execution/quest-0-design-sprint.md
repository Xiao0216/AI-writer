# Quest 0 设计冲刺

## 目标

在任何开发开始前，先完成最小可用设计交付。

## 必交付物

### 1. Figma 文件结构

必须包含以下页面或分区：

- 登录
- 项目控制台
- 项目真相区
- 章节控制页
- 会员页
- 马拉松控制页

### 2. 低保真线框

必须覆盖：

- 首页
- 项目页
- 章节页

### 3. 高保真关键页面

至少覆盖：

- 登录页
- 项目控制台
- 项目真相区
- 章节控制页
- 会员页

### 4. 交互状态清单

必须写清楚这些动作的：

- loading
- success
- error

动作范围：

- 登录
- 创建项目
- 生成控制卡
- 完成章节
- 会员升级

## 验收动作

### PM 检查动作

- [ ] 打开 Figma 文件
- [ ] 检查页面结构是否完整
- [ ] 检查关键页面是否有高保真稿
- [ ] 检查交互状态是否被定义

## 设计原则

- 首页必须像“控制台”，不能像普通作品列表
- 章节页必须像“工作台”，不能像普通编辑器
- 会员页必须像“能力说明页”，不能像单按钮支付页
- 所有状态必须能支持后续黑盒验收
