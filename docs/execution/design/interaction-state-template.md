# 交互状态表模板

> 用法：
> - 每个关键动作都要填这张表
> - 讲师、助教、学生、考官都按同一张表对齐

---

## 动作名称

例如：`开发环境快捷登录`

## 入口位置

例如：`登录页 / 开发环境登录区 / 主按钮`

## 前置条件

例如：

- 用户已输入 openid
- API 环境已启动

## Loading 状态

- 页面出现什么
- 按钮是否禁用
- 是否显示 loading 文案或转圈

## Success 状态

- 页面跳转到哪里
- 是否出现成功提示
- 哪个区域的数据应变化

## Error 状态

- 错误文案出现在哪里
- 页面是否保留当前输入
- 用户下一步可以做什么

## PM 黑盒检查动作

- [ ] 我点这个动作后，页面是否出现 loading
- [ ] 成功时，页面是否发生明确变化
- [ ] 失败时，页面是否出现明确提示

---

## 建议必须填写的动作

- 开发环境快捷登录
- 微信扫码登录
- 新建项目
- 保存项目真相文件
- 生成控制卡
- 生成正文初稿
- 保存正文
- 完成本章
- 开通会员
