# 设计交付日志

> 用法：
> - 每次 Quest 0 有交付更新时，学生都要补一条记录
> - 这个日志用于帮助 PM 和助教确认设计是否真的在迭代，而不是一次性截图交差

---

## 记录模板

### 日期

`YYYY-MM-DD`

### 本次完成内容

- 完成了哪些页面
- 完成了哪些状态
- 完成了哪些 Figma Page

### 本次修正内容

- 修正了哪些结构问题
- 修正了哪些命名问题
- 修正了哪些主次问题

### 当前未完成项

- 还缺哪些页面
- 还缺哪些状态
- 还缺哪些说明

### 下一步

- 下一步要完成哪一个 Quest

---

## PM 快速检查方式

- [ ] 设计日志至少存在一条记录
- [ ] 日志内容能对应到 Figma 文件中的真实变化
- [ ] 未完成项不是空泛口号，而是明确页面或状态
