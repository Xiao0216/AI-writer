# Quest 0.4 关键页面设计规格

## 页面 1：登录页

### 核心任务

让用户明确知道有两种入口：

- 真实微信登录
- 开发环境快捷登录

### 设计细节要求

- 微信登录按钮必须是页面视觉主 CTA 之一
- 开发登录区必须被明确标识为开发用途
- 失败提示必须有固定位置，不能漂浮不定

## 页面 2：项目控制台

### 核心任务

告诉作者“现在这本书最该推进什么”。

### 设计细节要求

- 首屏突出推进状态和提醒，而不是作品封面
- 热度、提醒、最近写回必须在首屏可见
- 项目列表只能作为控制台一部分，不得霸占首屏

## 页面 3：项目真相区

### 核心任务

让作者稳定访问标准控制文件。

### 设计细节要求

- 左侧或顶部必须有固定导航
- 当前激活文档必须有明显选中态
- 文档内容区必须适合长文本阅读和编辑

## 页面 4：章节控制页

### 核心任务

让正文创作和控制动作同时发生。

### 设计细节要求

- 正文区最大、最稳定
- 控制卡必须始终可见或易展开
- 切片区必须服务当前章，不得淹没正文
- 保存、生成、完成章节这些关键动作必须有固定位置

## 页面 5：会员页

### 核心任务

说明为什么付费，以及不同版本解锁什么能力。

### 设计细节要求

- 先讲能力，再讲价格
- 免费版、专业版、马拉松版必须有清晰对比
- 如果存在支付二维码区，也不能压过能力说明本身
