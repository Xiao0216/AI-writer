# Quest 0.1 Figma 文件搭建模板

## 本关目标

建立一个可维护、可评审、可开发对照的 Figma 文件结构。

## 学生必须创建的页面分区

### Page 1：00-Foundations

用于放：

- 颜色 Token
- 字体层级
- 间距规则
- 基础按钮与输入框样式

### Page 2：01-Wireframes

用于放低保真线框：

- 登录页
- 项目控制台
- 项目真相区
- 章节控制页
- 会员页
- 马拉松控制页

### Page 3：02-HiFi-Core

用于放高保真关键页面：

- 登录页
- 项目控制台
- 项目真相区
- 章节控制页
- 会员页

### Page 4：03-Interaction-Spec

用于放交互状态和流程：

- 登录
- 创建项目
- 生成控制卡
- 完成章节
- 升级会员

## 命名规范

每个画板命名必须包含：

- 页面名
- 状态名

正确示例：

- `Login / Default`
- `Login / Error`
- `Dashboard / Default`
- `Chapter Workspace / With Control Card`

错误示例：

- `首页`
- `页面1`
- `最终版2`
- `新的新的登录页`

## PM 黑盒验收动作

- [ ] 打开 Figma 文件，能一眼看到 4 个 Page 分区
- [ ] 检查关键页面是否都已经有占位或正式稿
- [ ] 抽查一个画板名字，命名能读懂状态和用途

## 助教提示

- 如果学生把所有页面堆在一页里，必须打回重做
- 如果学生只画首页，视为未完成
- 如果学生命名混乱，后续开发会直接失控，必须当场纠正
