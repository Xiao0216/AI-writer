# Quest 0.2 低保真线框检查清单

## 本关目标

先用线框图确认信息架构、视觉主次和用户路径。

## 每页必须体现的结构

### 1. 登录页

必须有：

- 品牌区
- 微信登录入口
- 开发环境快捷登录入口
- 登录失败提示区

### 2. 项目控制台

必须有：

- 当前推进区域
- 提醒区
- 情节线热度区
- 最近写回摘要区
- 项目列表区

### 3. 项目真相区

必须有：

- 文件导航区
- 当前文档工作区
- 文档状态或保存区

### 4. 章节控制页

必须有：

- 正文编辑区
- 控制卡区
- 人物 / 主线 / 伏笔切片区
- 状态反馈区

### 5. 会员页

必须有：

- 版本差异区
- 权益说明区
- 开通入口区

## 用户路径检查

学生必须能口头讲清楚这条路径：

1. 用户先登录
2. 登录后看到控制台
3. 从控制台进入项目
4. 在项目里进入章节页
5. 在章节页完成写作与控制

## PM 黑盒验收动作

- [ ] 抽查 3 张线框图，能立刻说出该页用途
- [ ] 要求学生口头讲主链路
- [ ] 检查章节页是不是“工作台”，不是单编辑器

## 打回标准

出现任意一条，直接打回：

- 首页像普通作品列表
- 章节页没有控制区
- 会员页只有付款按钮没有能力分层
- 项目真相区没有稳定导航结构
