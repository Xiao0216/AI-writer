# 文枢AI《项目全栈落地闯关大纲》第一层 Quest Map

> 这是正式的一层大纲文档。
>
> 使用顺序：
> - 产品手册：定义产品方向
> - 第一层 Quest Map：定义闯关路径
> - 第二层教案：定义提交物、卡点、评分
> - 第三层课程包：定义 Figma 设计前置、讲师演示与救火方式

## 阶段 0：Figma 设计定稿
*在任何编码之前，先把信息架构、关键页面和核心交互定下来。*

#### 🟢 Quest [0.1]：建立 Figma 项目文件与页面结构
* **🎯 关卡目标**：创建课程统一的 Figma 文件，并按产品结构拆好页面。
* **🛠️ 推荐工具/轮子**：`Figma`, Section, Page 命名规范。
* **🚫 边界限制 (防超纲)**：本关只做文件结构，严禁开始高保真视觉设计。
* **✅ 考官验收标准 (PM Checklist)**：
  - [ ] 打开学生的 Figma 文件
  - [ ] 能看到按产品模块拆分的页面结构
  - [ ] 至少有 `登录 / 控制台 / 项目真相区 / 章节控制页 / 会员页 / 马拉松控制页`

#### 🟢 Quest [0.2]：完成低保真信息架构草图
* **🎯 关卡目标**：用线框图确认主链路页面结构与用户路径。
* **🛠️ 推荐工具/轮子**：Figma Wireframe kit, Auto Layout。
* **🚫 边界限制 (防超纲)**：本关只画线框，不做配色和组件细节。
* **✅ 考官验收标准 (PM Checklist)**：
  - [ ] 查看首页、项目页、章节页线框
  - [ ] 能一眼看出页面分区和主次关系
  - [ ] 学生能口头讲清主链路：登录 -> 创建项目 -> 创建章节 -> 写作 -> 完成章节

#### 🟢 Quest [0.3]：定义设计系统基础 Token
* **🎯 关卡目标**：建立基础颜色、字体、间距、圆角、阴影规则。
* **🛠️ 推荐工具/轮子**：Figma Variables, Styles。
* **🚫 边界限制 (防超纲)**：本关只做基础 token，不做完整组件库。
* **✅ 考官验收标准 (PM Checklist)**：
  - [ ] 查看颜色和文字样式命名
  - [ ] 抽查两个页面，视觉风格一致

#### 🟢 Quest [0.4]：产出高保真关键页面设计稿
* **🎯 关卡目标**：完成关键页面的高保真设计稿，供后续开发对照实现。
* **🛠️ 推荐工具/轮子**：Figma Components, Auto Layout, Interactive Components。
* **🚫 边界限制 (防超纲)**：只覆盖关键页面，先不追求边缘页全量补完。
* **✅ 考官验收标准 (PM Checklist)**：
  - [ ] 查看高保真设计稿
  - [ ] 至少包含 `登录 / 项目控制台 / 项目真相区 / 章节控制页 / 会员页`
  - [ ] 页面可见布局、层级、状态，不是线框草图

#### 🟢 Quest [0.5]：定义交互规格与状态清单
* **🎯 关卡目标**：把关键交互的 loading / success / error 状态先定义清楚。
* **🛠️ 推荐工具/轮子**：Figma Prototype, 注释说明。
* **🚫 边界限制 (防超纲)**：本关不写实现方案，只定义交互与状态。
* **✅ 考官验收标准 (PM Checklist)**：
  - [ ] 查看交互注释和状态清单
  - [ ] 登录、创建项目、生成控制卡、完成章节都有状态定义

## 阶段 1：脚手架与统一测试环境

  - [ ] 能看到 `dev / build / lint / test` 入口

#### 🟢 Quest [1.2]：跑通前端空白站点
* **🎯 关卡目标**：让 `apps/web` 能独立启动，并显示一个最小可见页面。

  - [ ] 打开 `http://localhost:3000`

#### 🟢 Quest [1.3]：跑通后端健康检查服务
* **🎯 关卡目标**：让 `apps/api` 能独立启动，并暴露健康检查接口。

  - [ ] 执行 `curl http://localhost:3001/health`

  - [ ] 执行 `docker compose -f infra/docker-compose.yml up -d`
  - [ ] 执行 `docker compose -f infra/docker-compose.yml ps`
  - [ ] 能看到 `postgres` 和 `redis` 正在运行

#### 🟢 Quest [1.5]：建立统一测试环境入口
* **🎯 关卡目标**：建立前后端统一的 `test` 环境入口。
* **🛠️ 推荐工具/轮子**：`.env.test`, `dotenv-cli`, `npm scripts`。
* **🚫 边界限制 (防超纲)**：本关不联调业务页面。
* **✅ 考官验收标准 (PM Checklist)**：
